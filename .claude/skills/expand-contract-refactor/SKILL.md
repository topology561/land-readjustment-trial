# 寬幅重構：expand–contract 三段式

## 何時用（觸發條件）

**血脈遍及多檔的機械性改動**，典型：

- **去錨**（泛化波）：碼內字面錨 → fixture 資料檔／現算。
- **改共用符號**：函式改名/改簽章、`_WF_NS_NAMES` 增刪、session 鍵改名。
- **換資料來源**：讀 A 快照鍵 → 讀 B 鍵；讀碼內常數 → 讀查表。
- **搬移單一真相源**：同式散在多檔（#20 四處同改家族）→ 收斂為一函式。

判準：**若「改到一半」會讓 `run_all` 紅、或讓兩處同式脫鉤，就屬本 skill 管轄。**

## 為何不能一次改完

一次性大改的失敗模式（本專案已付學費）：

- 改 N 處、漏第 N+1 處 → **同式脫鉤**（`failure-archaeology` 之「同機制病灶散在多檔·『找到就以為找齊』」「同類 bug 藏多個獨立消費端」兩課）。
- 紅了之後**無法二分**：新錯與舊錯混在同一 diff，無從判「是改壞了還是本來就紅」。
- 刪舊形態後才發現有呼叫端還在用 → 只好緊急復原，或更糟：**加 fallback 把它接回去**（直接違反 `no-silent-fallback`）。

## 三段式

### ① expand（擴張：新舊並存）

- 新形態（如 fixture 資料檔／現算函式）**加進來**，舊形態（碼內字面錨）**原地不動**。
- **必加「兩者同值」斷言**：新形態算出的值 == 舊字面值，逐項比對，不同即 loud raise。
  - 此斷言是 expand 段的**唯一產出價值**——它證明新形態忠實複製舊行為。
  - 斷言**留到 contract 段才刪**，不是加完就拿掉。
- `run_all` **全綠才前進**。此時行為**零變更**（新形態尚無人消費）。

### ② migrate（遷移：逐檔改讀新形態）

- **逐檔／逐批**把消費端從舊形態改讀新形態。**分批**，一批一 commit。
- **每批 `run_all` 保綠**才做下一批。批次大小以「紅了能一眼定位」為準。
- 同值斷言仍在 → 任一批若改錯，斷言或 `run_all` 立刻咬。
- **禁**在本段順手改行為。遷移就只是換來源，值不許動。

### ③ contract（收縮：刪舊形態）

- **先證零殘留**：grep 舊符號／舊字面值，確認**無任何呼叫端**。
  - grep 要搜**多個變體**（不只精確字串）——同一慣用法常有寫法差異。
  - 「grep 到兩處」不等於「找齊」；掃呼叫圖**與**掃程式碼慣用法變體，兩者都做。
- 確認零殘留後，**整段刪**（連同同值斷言）。**禁**留舊分支、禁留 fallback、禁留孤兒常數。
  - 孤兒常數（僅定義、零讀取）也要刪——留著＝日後被誤引為權威。
  - 刪除時原地留一行註解指向新正典位置。
- 刪後 `run_all` **仍綠**。

## 與既有紀律的銜接

- **`no-silent-fallback`**：contract 段刪舊形態後，新形態**缺鍵必 loud raise**，嚴禁「找不到就退回舊值／預設值」。遷移期間的並存是**有斷言看守的暫時態**，不是 fallback；兩者差別在於：並存有「同值斷言」證明等價且**有明確拆除點**，fallback 沒有。
- **`wave-discipline` 凍結欄框架**：
  - **expand／migrate ＝行為保持波（求一致）**：凍結欄＝一切數值輸出；可變欄＝資料來源。驗收＝**逐格 diff 全同**。「不變」證明沒污染。
  - **contract ＝亦為行為保持波**：刪除前後**逐格 diff 全同**；若 contract 後有任何值變動，即代表舊形態仍在被消費（遷移沒做乾淨），**不是**「刪對了」。
  - 一波一事：expand/migrate/contract 各自成波或成批，**禁**與功能變更混在同一 commit。
- **`validation-runbook`**：每段的「保綠」以其列之驗收命令為準，不自創判準。
- **`stop-conditions`**：同值斷言破 → 停機查，**禁**調寬斷言使其變綠。

## 反面樣式（見到就擋）

| 反面 | 為何錯 |
|---|---|
| 一個 commit 同時新增新形態、改所有消費端、刪舊形態 | 紅了無法二分；等於沒有 expand/contract |
| expand 段不加同值斷言 | 新形態沒被證明等價，migrate 等於盲改 |
| 同值斷言不過 → 調寬容差／改期望值 | 「把綠當過」；斷言的全部價值即在它會咬 |
| contract 前沒 grep 就刪 | 漏掉呼叫端 → 緊急復原或補 fallback |
| 舊常數「先留著以防萬一」 | 孤兒常數會被日後誤引為權威 |
| migrate 順手改了值 | 凍結欄破，無法區分「遷移錯」與「改對了」 |

## 檢查清單（動工前逐項打勾）

- [ ] 已列出**全部**消費端（grep 多變體＋呼叫圖），並記下數量作為 contract 段的核對基準。
- [ ] expand 段的同值斷言寫好了，且**會咬**（先故意改一個值驗證它紅）。
- [ ] 分批計畫已定：每批範圍、每批的 `run_all` 保綠點、一批一 commit。
- [ ] contract 段的零殘留 grep 命令已寫進計畫（含變體清單）。
- [ ] 已宣告凍結欄／可變欄，且三段皆為「行為保持波」尺標。
