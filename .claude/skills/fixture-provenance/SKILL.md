# Fixture 出處紀律（套套邏輯測試禁令）

## 核心禁令

> **嚴禁**「跑一次新碼 → 把它吐出來的數字寫進期望值 → 測試全綠」。

那不是測試，是**同義反覆（tautology）**：你只證明了「程式等於它自己」。
一旦新碼本身就是錯的，這種 fixture **永遠不會紅**，而且會把錯誤**凍結成正典**——
日後任何人想修對，反而會被這個假 fixture 擋下來。

這是「**把綠當過**」的 fixture 版（見 `CLAUDE.md` 最高紀律節「🔒 自評 gate 綠不可替代獨立復現」：
自跑 gate 綠是**最不可自我採信**之聲明，該類驗證之**全部價值即獨立復現**）。

## 合法的期望值來源（四類，逐值須註記其一）

| 來源 | 說明 | 註記形式（範例） |
|---|---|---|
| **倉內既有錨** | 轉 fixture 前碼裡／baseline 裡已存在、且**曾被獨立復現過**的值 | `來源：verify/baselines/wf/f4/F.4_七五雙出口_退縮0m.csv 欄「G(a′)輪0(㎡)」` |
| **KL 授權精度值** | 域權威給定之數（含其授權的精度位數） | `來源：KL 裁定 2026-07-11（第11號公報 P.4 公設地後價）` |
| **104 手冊／法規** | 手冊圖表、辦法條文推導 | `來源：104 手冊 圖17 p.192–193 道路用地分配原則` |
| **PROVENANCE 檔** | baseline 重烤時記錄的出處鏈 | `來源：verify/baselines/wf/f0/PROVENANCE_F0.md` |

**不在此四類者一律不得當期望值。** 存疑即上呈，禁自決（見 `stop-conditions`）。

## 逐值註記（強制）

每個 fixture 值**旁邊**要能回答：「這個數字**不是**從被測程式來的，是從哪來的？」

```python
# ✅ 有出處
EXPECT_MINA_QU = 114.07   # 來源：run_verification.py 正典閘 MinA_區 推導斷言
                          #       ＝round(R4 深度 32.59 × min_width 3.5, 2)；辦法 §3 第三位四捨五入

# ❌ 無出處（等同套套邏輯，見到即擋）
EXPECT_MINA_QU = 114.07   # 跑出來就是這個
```

註記寫**倉檔路徑**或**裁示編號**；禁寫「實測」「跑出來的」「之前就是這樣」。
**禁憑記憶出艙**（`CLAUDE.md`：數字錨一律引倉檔路徑，與倉態衝突以倉為準）。

## 錨值轉 fixture 的正確作法

錨值轉 fixture **本身是寬幅重構**，走 `expand-contract-refactor` 三段式：

1. **expand**：fixture 檔新增，值**逐項對拍舊碼內字面錨**（新舊同值斷言）。
   - 此時期望值的出處＝**舊錨**，而舊錨的出處要能上溯到上表四類之一。
   - 若上溯不到 → **該錨本身就是套套邏輯遺留**，列待裁、禁直接沿用。
2. **migrate**：測試改讀 fixture，`run_all` 保綠。
3. **contract**：刪碼內字面錨，逐格 diff 全同。

⚠️ 轉檔**不會**讓一個沒出處的值變得有出處。搬家不等於洗白。

## Fixture 必須「會咬」

寫完 fixture 後**先驗證它會紅**：故意把被測值改壞一點，確認測試翻紅；再改回。
不會咬的 fixture ＝ 假綠（見 `failure-archaeology` 之「以殘餘定閘寬·『閘全綠』的效力上限＝閘寬依據強度」，
以及「單點錨對『未被通式參數化之維度』零舉證力——對的單點掩護錯的全域」）。

同源陷阱：**期望值與被測值若走同一段程式算出，斷言恆成立**（相消）。
fixture 的兩邊必須**真異源**——一邊是被測引擎、另一邊是手算／倉檔／獨立實作。

## 泛化波專用提醒

去錨改造時，A 類案例字面值轉入「UC9898 黃金 fixture」：

- 黃金 fixture 的期望值＝**去錨前的舊值**，其出處＝舊 baseline／PROVENANCE。
  這是合法的（來源＝倉內既有錨），**前提是**該舊值本身經過獨立復現。
- **但**：泛化後若引擎行為理應改變（如換案例、修正錯誤實作），黃金 fixture 會紅——
  此時**禁**改期望值使其變綠，須先判「是泛化改壞了，還是舊錨本來就錯」，
  依 `stop-conditions` 停機上呈。
- 新案例的 fixture **不得**由「跑新引擎一次」產生。新案例期望值須有其自己的四類出處
  （該案的 KL 裁示／手冊推導／人工試算）。

## 檢查清單

- [ ] 每個 fixture 值都有**倉檔路徑或裁示編號**級的出處註記。
- [ ] 沒有任何值是「跑被測程式一次回填」得來。
- [ ] fixture 已驗證**會咬**（故意改壞 → 紅 → 改回）。
- [ ] 期望值與被測值**異源**，非同一段程式的兩次呼叫。
- [ ] 舊錨轉 fixture 者，舊錨的出處可上溯至四類之一；上溯不到者已列待裁。
