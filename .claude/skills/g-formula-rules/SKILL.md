# G 公式與負擔規則（KL 已鎖）

## 公式鎖定形
```
G = (a×(1−A×B) − Rw×F×l_side − S×l_front)×(1−C)
```
- **B、C 是全案常數**——不因街廓 ΣRw 而逐塊變動。任何「按塊調 B/C」的想法都是錯的。
- **F ＝ 街角地第 1 筆面臨側面道路之長度 ＝ SIDE_LINE 全長**（畫到未截角理論角；見 cad-layer-semantics）。
- 街角 G-gate 用估值：`G估 = a×(1−重劃總負擔率)+0.5`；資格＝ `G估 ≥ 該端街角最小分配面積(range 面積)`。

## 負擔總面積的長度來源
- 正面道路負擔總面積＝**FRONTLINE 長**×正面尺度（按面臨正街寬查表）。
- 側面道路負擔總面積＝**SIDE_LINE 長**×側面尺度（按面臨側街寬查表；側街取半，見 l1 定義）。
- 兩者都用**未截角全長**——這就是 CAD 線畫到尖角的原因。

## 池（調配池）規則
- ΣRw < 100% 時，缺口由**該街廓的池吸收**（B+C 的未區分殘量）——**不歸因**到任一成分。
- 池**預設中央擺放**；僅在需要最大化 ΣRw 時最小幅度移動。後續波次從 ΣRw 較低側向內消化池。
- **通用分配型**：「兩端向中央、池居中」——適用一切街廓型。
- **兩階段時序**：第一階段池的產生是 G 公式扣減的數學必然；第二階段消化可能歸零。**逐街廓獨立最大化是正確的**（最大化各街廓第一階段池備、供後續消化）。
- 街角全數不合格 → 第 1 宗＝抵費地、面積鎖定＝range 面積；**中央池仍存在**（角落抵費地＝池面積重定位、非額外面積）——守恆 ΣG+池=街廓 必須成立。
- **W-D.2 已知接線缺口**：`_forced_offset_map` 之 `left/right_corner_min_area` 從不賦值 → 角落抵費地面積以 0 流入 `_spatial_order_parcels_v2` → 池重定位守恆**尚未接線**（W-D.2 §3 必做項）。

## 參數紀律
- 退縮、法定最小寬等是**案例/街廓級參數**，不是系統常數——禁止靜默預設（見 no-silent-fallback）。
- **畸零地最小寬/深查找＝二維表：使用分區 × 面臨道路寬度**（花蓮縣畸零地使用規則附表；正面路寬為使用者輸入）。本案住宅區×12m 與 ×8m 恰同檔 3.5m——**巧合非常數**（KL v2 定案 2026-07-05）。
- **已知缺口（修復屬 UI/接線波）**：`f3L_sb_rows_by_label` 全 app 無人寫入 → v12 內法定最小寬**恆走 3.5 預設**。任何（分區×路寬）查表值 ≠3.5 的案例即觸發（同案內臨寬路街廓亦可）：T 錯→range 錯→G-gate 錯。修復＝建「(分區,路寬)→最小寬/深」查表接線，資料源＝畸零地規則附表。
- **`or 3.5` falsy-zero 陷阱**：退縮 0m 是合法值，`x or default` 會把 0 吃掉——一律用 `x if x is not None else default`。
- `has_left/has_right` 判定用 **SIDE_LINE 存在與否**，不是路寬（路寬可為 0 而截角存在，如 R1 右）。

## W-D 波次的預期輸出（不是 bug）
G 值變動、ΣRw 變動是 W-D（位次定案）波次的**預期產出**——停機條件只有：不變量破 / J 下降 / 守恆破（見 stop-conditions）。
