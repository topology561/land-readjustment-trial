# W-A 四決策裁示（KL 已拍板，Claude Code 依此動工）

W-A 範圍：四欄面積架構 + 0.1m 垂距門檻 + DXF 比例分攤 + 停污染。
依 CLAUDE.md 與 docs/配地計算總規格_v3.md 施工。

## 決策 1：選 B，附帶條件必須一起做

G 公式的 a 在 W-A 階段只讀 `分攤登記面積_m2`，先不接 a' 累加器，
a' 的接線留到 W-F 與調配引擎一起做。接受「Tier 1 a' 貢獻在 W-A 到 W-F
之間暫時失效」這個過渡代價。

**附帶條件（務必做到，否則 W-F 踩雷）**：選 B 不代表 W-A 完全不碰
`面積_m2`。必須在 W-A 就把 `面積_m2` 清成乾淨的累加器起點（重設為 0），
並停掉所有對它的污染寫入（見決策 2）。W-A 結束時 `面積_m2` 必須是乾淨、
歸零、無污染殘留的累加器，只是 G 公式和 a' 累加的接線暫時不啟用。

## 決策 2：停 E-0.2 污染累加

`分攤登記面積_m2` 機制已天然保證「同原地號各暫編加總 = 登記面積」，
E-0.2 的帳面累加變成冗餘。W-A 停掉 E-0.2 的 `面積_m2 += sub_area`
（line ~3859），ghost 只保留「幾何留原位、染灰渲染」的視覺功能，
不再動任何面積欄位。

同步處置孤兒增配（line ~6332 `面積_m2 += share`）：它是 a' 性質的
舊版調配實作（W-F 會被新調配引擎取代），W-A 先讓它不污染 `面積_m2`。

## 決策 3：0.1m 垂距門檻與 _is_sliver_polygon 並存

0.1m 垂距先判（界址點到 BLOCK 邊界垂距全 < 0.1m → 不切暫編、
整筆按登記面積顯示），sliver 當第二道。兩判據抓不同情況
（垂距抓「貼邊界圖解誤差」、sliver 抓「細長碎片」），只留一個會漏網。
雙重剔除不是問題——被任一道剔除的都是真不該編暫編的碎片。

## 決策 4：驗收看 `幾何面積_m2`

W-A 驗收 = `幾何面積_m2` 欄位對齊 DXF 真值：
628-18(2) 應顯示 383.46（不再是 481.08）、628-45(1) 應 92.68（不再是 623.66）。
R5 全部 12 塊真值見 CLAUDE.md 驗收對拍基準。
5 塊小切角（628-53(1) 2.79㎡、628-32(1) 5.43㎡ 等）依 0.1m 門檻正確處理。
