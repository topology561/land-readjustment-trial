## 〇、根本概念（先讀）

### 概念 1：試分配 vs 實際作業

- **現階段（試分配）**：BLOCK 間未逕為分割，有「跨街廓暫編」，需 DXF 比例分攤登記面積。
- **未來（實際作業）**：BLOCK 間先逕為分割，暫編僅存在於分配街廓內。
- 本規格為試分配設計。需合議制審議的調配，試分配時**以本規格的調配原則自動跑預設**，
  正式作業時由使用者依合議結果以介面覆寫（見八-6）。

### 概念 2：兩個虛擬固定多邊形（勿混用）

| 多邊形 | 寬度定義 | 用途 |
|---|---|---|
| **評比範圍** | W_eval = 退縮 + 畸零地最小寬 | 找重劃前宗地交集、算街角評點指數 |
| **負擔範圍** | W = 18m（固定不分區）| 算範圍內各分配宗地的側街負擔 Rw·F·l₁ |

兩者畫定後不隨 G 值變動；地界線（依 G 值平移）則隨 G 值移動。

### 概念 3：守恆式（v3 鐵律，全部調配規則掛在這上面）

```
任何時刻，對每個可建築街廓 Ri：
  Σ G(街廓內所有分配地) + 調配池(Ri) = 街廓 DXF 面積(Ri)
```

任何 a' 調配（同歸戶合併或無同歸戶調配）都讓某些地的 G 變大，
增量必然來自調配池——**沒有「不消耗調配池」的調配**。

### 概念 4：終極驗收標準

重劃完成後，分配於可建築街廓內的**每一筆土地**（含留設的調配池若有）
均須符合該街廓最小建築面積，才可建築。任何調配結果違反此標準即無效。

### 概念 5：CAD 圖層規範（v3.1 定稿，使用者繪圖 SOP 依此）

| 圖層 | 必然性 | 語意 | 方向 |
|---|---|---|---|
| **FRONT_LINE** | 必有（可建築街廓一定臨路）| 正面臨路線 | **起點 p1 = 左、迄點 p2 = 右**（方向約定）|
| **SIDE_LINE** | 可有可無（不臨側街則無）| 側街線；有則該側有街角地 | 端點接 FRONT_LINE p1→左街角、接 p2→右街角 |
| **BLOCK** | 必有 | **使用分區界線**（R↔RD、R↔G、RD↔G 等）| — |
| **BASELINE** | 每可建築街廓必畫一條 | **屁股線**（街廓內側分配基準線）| 不需方向（只取方向軸、取位置）|
| **宗地分配線** | 每可建築街廓必畫一條 | **定義該街廓地界線方向**（= 地界線本身的方向）| 純方向（位置不重要、只取方向）|
| **CENTERLINE** | 道路用地 | 道路中心線（RD 調配切割用）| — |

**宗地分配線定稿**（v3.1 新增，解決弧形/不規則街廓）：
- 「宗地分配線」=「地界線」**二詞同義，是同一條線的概念**：
  使用者在街廓隨意位置畫一條宗地分配線，定義這個街廓所有地界線的方向。
- 每一筆宗地依其 G 值（應分配面積 = 重劃後扣公設地負擔+費用負擔），
  **沿宗地分配線方向平移**到對應位置；平移後的線即該筆與下一筆的地界線。
  → 方向由使用者給（宗地分配線），位置由 G 值算（平移量），同一族平行線。
- 程式只取宗地分配線「方向」（使用者畫的位置不重要，實際位置由各筆 G 值決定）。
- **不管 FRONT_LINE / BASELINE 是否彎曲**——地界線方向只跟宗地分配線，故弧形街廓亦適用。
  R1、R2 可各畫不同方向的宗地分配線。

**BASELINE 定稿**：劃設位置 = 屁股線本身（**不再平移 0.5m**，舊 0.5m 取消）。
- 大街廓需切分（如住宅區太寬深 → R4/R5/R6）：BASELINE 畫在切分位置，
  兼任「分配線（同分區內街廓分隔）」。跨 BASELINE 的土地**不逕為分割**。
- 小街廓不需切分：BASELINE 重疊畫在 BLOCK 屁股線上。
- **分配線不另設獨立圖層**——由 BASELINE 兼任。

**三種暫編來源**：
1. BLOCK 線（使用分區界）切出 → **逕割性質**暫編（未來逕為分割成真實子地號，
   各自有登記面積）。
2. BASELINE（屁股線/分配線）切出 → **分配性質**暫編（不逕割，試分配作業用，
   用分攤登記面積）。
3. 地界線（依 G 值平移）細分 → 分配性質（同 2）。

**街角地非必然**：只有「有 SIDE_LINE」的街廓才有街角地。無 SIDE_LINE 的街廓
（如三面臨公園/市場/停車場、僅北面臨路）無街角地——跳過街角評點與 Rw 側街負擔，
土地直接依 FRONT_LINE 投影位次排。

### 概念 6：推進方向 + 調配池兩階段（v3.1 定稿）

**推進方向（一律由 FRONT_LINE 定，不依賴 SIDE_LINE）**：
- 雙側街角：兩端各從街角往中間推。
- 單側左街角：從 p1（左）往 p2（右）。單側右街角：從 p2 往 p1。
- 無街角：兩端各依原位次投影往中間推（FRONT_LINE p1↔p2）。
**「兩端往中間排、調配池居中」是通則**（三種街廓皆是），差別僅在端點
要不要算街角評點與 Rw。

**調配池兩階段時序（勿混淆）**：
- **階段一（產生）**：街廓內重劃前土地兩端往中間排、逐筆算 G。因 G 公式扣了
  負擔（公設地負擔% + 費用負擔%），ΣG **必然** < 街廓 DXF 面積 →
  中間**必然**留出調配池（≈街廓面積×C%相關）。**原位次分配後調配池必存在、必 > 0。**
- **階段二（消耗）**：三順位+六級調配把外來 a' 灌入調配池消耗之，
  終止於吃光歸零 或 留最小建築面積（不留中間態）。
- ⚠️ Claude Code 勿把「階段二的歸零」誤植到階段一——原位次分配時**不判斷**
  要不要留調配池（必留），調配階段才消耗。若階段一就 ΣG > 街廓面積，
  屬街廓劃設錯誤、回報 KL，非正常流程。

## 一、四欄面積架構

| 欄位 | 性質 | 來源 / 算法 | 用途 |
|---|---|---|---|
| `登記面積_m2` | 唯讀 | Tab 1 匯入，原地號法定面積 | 權利大數，分攤母數 |
| `幾何面積_m2` | 唯讀 | DXF ∩ BLOCK 真實 polygon 面積 | B/C值、區段彙總、平均地價權重、街廓對齊 |
| `分攤登記面積_m2` | 唯讀 | 登記面積 × (該暫編幾何 / Σ同原地號各暫編幾何) | G 公式分子 a 的原生部分 |
| `面積_m2` | 動態 | a' 折算累加器 | 承接調配進來的 a' |

**G 公式分子**：`a_for_G = 分攤登記面積_m2 + Σ(調配進來的 a')`
**鐵律**：所有調配都以重劃前面積 (a + a') 重新進 G 值計算——
不是把若干 G 相加，是先合併面積、再一次算 G'。

### 分攤公式

原地號 1 登記 100，DXF 切成 1(1)/1(2)/1(3) 各 20/30/40（總和 90）：
分攤登記面積 = 100×20/90、100×30/90、100×40/90 = 22.22/33.33/44.45（總和=100）。
GIS 顯示用分攤登記面積，非 DXF 原值、非污染累加值。

### 圖簿不符由調配池吸收

DXF < 登記 → 調配池變小；DXF > 登記 → 調配池變大。守恆式恆成立。

## 二、DXF 軌 vs 登記軌職責分工

| 計算項目 | 用哪軌 | 法理 |
|---|---|---|
| G 公式分子 a / a' | 登記軌（分攤登記面積）| 市地重劃實施辦法§29 |
| B 值、C 值分母 | DXF 軌 | 全區負擔母數用實測 |
| 平均地價 P_average 面積權重 | DXF 軌（幾何面積）| 空間加權 |
| 區段彙總（Tab 6）| DXF 軌 | 估價區段是實體分區 |
| 道路長度、街角範圍、W、S | DXF 軌 | 實體幾何 |
| 各 Ri 分配後土地面積加總 | = 街廓 DXF 面積 | 守恆式 |

## 三、暫編切割門檻：0.1m 垂距規則

逕割地籍線段界址點（不含與 BLOCK 邊界共用點）到 BLOCK 邊界垂距：
max(d_i) < 0.1m → 不切暫編、整筆按登記面積顯示；
max(d_i) ≥ 0.1m → 切暫編、DXF 比例分攤。
法理：1/500 圖解區、筆尖 0.2mm ≈ 實地 10cm = 同一點。

## 四、街角規定範圍多邊形（評比範圍）

### 構造程序（產出封閉多邊形，邊數不預設）

1. 取該側 SIDE_LINE 中點 M。（無 SIDE_LINE 的街廓無此步、無街角範圍）
2. 將該街廓的**宗地分配線方向平移到 M**（宗地分配線定義地界線方向，平移到 M
   後方向不變）。此為「沿宗地分配線方向」的基準線。
   （不再用「垂直於 BASELINE」——弧形街廓 BASELINE 彎曲無唯一垂直方向；
    改以宗地分配線方向為準，見概念 5。）
3. 從 M 沿宗地分配線方向往可建築用地方向平移 W_eval = 退縮 + 畸零地最小寬，得平移線 L'。
   方向正負判定：平移後靠該街廓質心側為正。
   畸零地最小寬 = `get_min_lot_size(分區, 正面臨路寬)['min_width']`。
4. 平移線 L' 與 SIDE_LINE、FRONT_LINE、BASELINE（屁股線）圍成封閉多邊形。
   ⚠️ 邊數不預設：直線街廓圍四邊形；弧形街廓 FRONT_LINE/BASELINE 為折線時邊數更多。
5. **扣除道路截角三角形**（三頂點 = FRONT_LINE×SIDE_LINE 理論交點
   + 截角斜邊在 FRONT_LINE 側端點 + 在 SIDE_LINE 側端點；屬道路用地）→ 最終多邊形
   （直線街廓五邊形、弧形街廓邊數更多；程式以通用多邊形運算處理，勿假設邊數）。

驗證基準：R5 左側 ≈ 146.50㎡（DXF 預埋圖層，退縮 0m）。
官方算式（手冊圖 7）：8.5m × 40m − 4㎡ = 336㎡（最小寬 5 + 退縮 3.5 = 8.5）。

### 取代 `_build_corner_polygon_B4`

整個重寫為 `_build_corner_range_v2`。`f3_current_pk_block` 未寫入 bug 一併修
（PK 迴圈呼叫前補 `st.session_state['f3_current_pk_block'] = _lbl`）。

## 五、負擔範圍多邊形（W = 18m）

構造同第四節但平移 18m、不扣截角。
**W（累積值）= SIDE_LINE 中點到該分配宗地「靠街角側地界線」的距離，沿宗地分配線方向量、
沿街廓遞增。** 操作：宗地分配線平移到 SIDE_LINE 中點，沿其方向推 18m 即 W=18m 範圍。
（地界線都平行宗地分配線，故「垂直地界線的垂距」= 沿宗地分配線方向量的距離。
弧形街廓亦適用，因量測基準是宗地分配線方向、非彎曲的 FRONT_LINE/BASELINE。）
W ≤ 18m 內的分配宗地扛 Rw·F·l₁；超出部分 Rw=0。

## 六、五步分配流程（原位次階段）

### 第 1 步：投影位次

重劃前宗地（不論面積多小，一律）投影到 FRONT_LINE 排序。
「原位次」= 位相關係（相鄰順序），非物理座標。例：5、3、1、2、8、7、9。

### 第 2 步：街角評點指數（依手冊圖 8）

交集面積 > 0 即納入評比。三項指數（各分母 = 參評各筆該項合計）：

| 項目 | 權重 | 分子 | 分母 |
|---|---|---|---|
| 臨正街角線 | 0.4 | 該宗地貼到道路截角斜邊的長度 | 截角斜邊全長 |
| 臨側街線 | 0.2 | 該宗地臨側街長度 | 各參評筆合計 |
| 跨占街角範圍面積 | 0.4 | 跨占面積 | 各參評筆合計 |

道路截角斜邊 = FRONT_LINE 與 SIDE_LINE 被截角切斷後兩端點連線。
⚠️ 分母是斜邊長，**不是最小建築寬度**（手冊範例兩者都是 5m，純屬巧合）。

官方對拍（圖 8）：1 地號 (5/5)×0.4+(25/37)×0.2+(100/300)×0.4=0.67；
2 地號 0.33 ⟹ 1 號勝。

### 第 3 步：定案位次 + 遞進解 G

#### 3a. 街角第 1 筆（逐名次試 G）

各側指數第 1 名起試 G；達街角規定範圍面積者為街角第 1 筆；
全未達 → 該側街角 = 抵費地（屬調配池之街角部分）。

#### 3a-1. 街角第 1 筆實際土地的形狀（不預設邊數）

街角第 1 筆實際分配後土地的形狀，由以下封閉多邊形圍成：
1. **SIDE_LINE**（該側側街線）
2. **FRONT_LINE 相關的段**（從 SIDE_LINE 端點，沿 FRONT_LINE 走到地界線交點為止；
   弧形街廓的 FRONT_LINE 若為折線、此段可能跨多段折線）
3. **第 1 筆與第 2 筆之間的地界線**（依 G 值反推位置、平行 ALLOC_LINE 方向）
4. **道路截角斜邊**（屬道路用地、不含入這筆地）

⚠️ **邊數不預設**：直線街廓、FRONT_LINE 單段、街角地不大時恰為五邊形（手冊圖 7 即此特例）；
但**弧形街廓**或 FRONT_LINE/BASELINE 在街角附近有多段折線、且**第 1 筆重劃前面積大**
（G 值反推出的地界線位置往內推得遠、跨多段折線）時，邊數會更多——可能六邊、七邊
或更多。程式以 shapely 通用多邊形運算處理（Polygon + 折線端點完整、difference 扣截角），
**勿假設邊數、勿硬編 5 個頂點**。

街角範圍（評比用虛擬多邊形，第四節）與街角第 1 筆實際土地，雖圍成方式類似
（皆 SIDE_LINE + FRONT_LINE 相關段 + 內側一條線 + 截角），但用途與內側線不同：
- 街角規定範圍：內側線 = 沿 ALLOC_LINE 方向平移 W_eval 的**虛擬平移線**（用於評比資格）。
- 街角第 1 筆實際土地：內側線 = 第 1 筆與第 2 筆**實際地界線**（由 G 值定位）。

⚠️ **兩條內側線方向同源、皆由 ALLOC_LINE 方向產生**：
- 虛擬平移線：沿 ALLOC_LINE 方向、距 SIDE_LINE 中點 W_eval。
- 實際地界線：沿 ALLOC_LINE 方向、距前一條地界線 = G 值反推的平移量。

這是整個街廓所有「方向性線」的一致性鐵律——**該街廓內所有地界線、街角範圍的
平移線、W 量測方向，全部以 ALLOC_LINE 方向為唯一基準**（v3.1 概念 5）。
程式勿讓不同用途的線用不同方向基準。

#### 3b. 其餘回歸投影位次

街角第 1 筆用指數選（特例），第 2 筆起回歸投影位次。

#### 3c. 遞進解 G（按位次遞進，非逐筆獨立）

**兩個量分清（W 與 S 量測方向不同，勿混）**：

| 量 | 量測方向 | 定義 | 用途 |
|---|---|---|---|
| **W**（宗地寬度）| **垂直宗地分配線**（沿其方向量）| 左地界線中點到本筆右地界線垂距；第 2 筆起 = 平移距離 | **查 Rw 側街負擔 + 比畸零地最小寬度判去留**（兩用途同一量）|
| **S** | **沿 FRONT_LINE** | 本筆兩條地界線交 FRONT_LINE 兩點距離 | G 公式正街負擔 S·l₂ |
| 最小建築寬度（常數）| — | 畸零地附表查表值 | 門檻，比 W |

⚠️ **判畸零地寬度用 W、不用 S**：宗地分配線斜交 FRONT_LINE 時，S（臨街面）會虛胖
（極端如兩地界線交 FRONT_LINE 成 170°/10°，平行四邊形狹長地，S 很長但 W 很窄、
實際難建築）。故「夠不夠最小寬度」一律看 W（垂直宗地分配線的真實寬度），不看 S。

```
W_前 = 0
按位次推進每筆：
  G_本筆 = [a(1−A·B) − Rw·F·l₁ − S·l₂](1−C)
    Rw 配額 = (R(W_本筆) − R(W_前))/100  ← 累積差額，rw_from_width 表
    W_本筆 = SIDE_LINE 中點沿宗地分配線方向到本筆右地界線距離（第2筆起=平移距離）
    S = 本筆兩地界線在 FRONT_LINE 交點距（沿 FRONT_LINE）
  反推地界線位置 → 得 W_本筆
  W_本筆（垂直宗地分配線寬度）< 最小建築寬度 → 此筆移出本街廓位次、列入第 1 調配順位（七-1）
  W_本筆 ≥ 18 → 之後各筆 Rw=0
```

Rw 累積驗證：47.64+23.47+23.33+5.56 = 100.00% ✓
> 現有 app.py（4843/4876/4909）逐筆獨立查 Rw 且僅街角筆扣——**錯**，重寫。

#### 3d. 最小分配面積定義（v3.1 統一）

**最小分配面積 = 街角規定範圍面積**（由 `_build_corner_range_v2` 算出的真實多邊形）。

也就是：第四節「街角規定範圍」與本節「街角最小分配面積」**指同一個量、同一套幾何構造、同一個值**，由「SIDE_LINE 中點 + ALLOC_LINE 方向法向平移（退縮 + 法定最小寬）+ 與 SIDE_LINE/FRONT_LINE/BASELINE 圍多邊形 + 扣道路截角」算出。

⚠️ **不得**用矩形近似公式 `(退縮 + 最小寬) × 街廓平均深度 − 截角` 計算——
該公式在不規則街廓會誤判（FRONT_LINE/BASELINE 微傾、或弧形折線時，矩形近似與真實多邊形差距可達數平方公尺；用錯門檻會讓 PK 第 1 名候選筆數錯估）。
此舊公式（曾以 `calc_corner_min_area` 實作）在 W-B 已移除、不留 fallback。

> 對拍依據：本案 R5 左側退縮 0m 時，真實多邊形面積 = 146.93 ㎡（v3.1 程式）
> ≈ DXF 預埋基準 146.50 ㎡（差 0.43 ㎡ 為 shapely 浮點精度）；
> 矩形近似公式給 150.39 ㎡（差 3.46 ㎡），不採用。

無街角街廓無街角範圍 → 無此值；該街廓「最小分配面積」改以該街廓「最小宗地寬度」
配上街廓深度估計（或由使用者另行設定，留待 W-D 訂規則）。

#### 3e. 調配池位置（兩端往中間、居中，三街廓通則）

所有可建築街廓（雙側/單側/無街角），土地兩端依各自原位次往中間排，
調配池**居中**（見概念 6）。調配池不扛側街負擔，居中自然避開兩端的 W=18m
（若該端有街角）。調配池居中且臨 FRONT_LINE 側 → 必臨路（它也是重劃後可建築土地）。

- 雙側街角：兩端各從街角往中間，例 `5｜3+3'｜1｜調配池｜4'｜2`。
- 單側街角：有街角端從街角排（算 Rw）、無街角端從原位次排（不算 Rw），居中。
- 無街角：兩端皆從原位次投影排（皆不算 Rw），居中。

調配池此時為「階段一產生」的必然結果（概念 6），尚未被消耗。
階段二調配（七節）才消耗它。

## 七、調配池定義 + 三調配順位（v3 核心重寫）

### 7-0. 調配池定義（取代舊「抵費地池」一詞）

```
調配池(Ri) = 街廓 DXF 面積(Ri) − Σ G(Ri 內得以分配土地)
           ≈ 街廓 DXF 面積 × (公設地平均負擔% + 重劃費用平均負擔%)
組成 = 街角抵費地（若有）+ 街廓中間調配池
用途 = 公設地負擔部分（本來就預留供調配）+ 費用負擔部分（標售自償，必要時亦可調配/增配）
```

注意：C% 全區統一、不分街廓大小 → 小街廓的調配池**可能先天 < 最小分配面積**。
**不是每個街廓都必須留設調配池**——可歸零；要留就必須 ≥ 該街廓最小分配面積（可建築）。

### 7-0a. 前置地基檢查（最先跑，調配前）

```
若 min(各街廓最小分配面積) > 重劃區總面積 × C%：
    警示「分配街廓需重新劃設」，停止，不進調配。
```
（全區調配池總量連一塊最低門檻的合格地都湊不出 = 規劃失敗，調配救不了。）

### 7-1. 第 1 調配順位：可建築街廓內未達分配面積者

依「**同歸戶 4 級合併分配**」處理（無需事前檢查調配池底線，吃到池盡為止）：

| 級 | 內容 |
|---|---|
| 級 0 | 同街廓同歸戶：小塊往**已達標**的最大塊集中合併 |
| 級 0' | **弱弱聯合**（手冊宗地分配原則(三)）：同街廓同歸戶**皆不達標**者，全部往其中「應分配面積較大者」（實作可用 G 值最大者）合併；合併後 G(Σa) 達標 → 以該最大塊的投影位次參與原位次分配。**不得合併分配於公設地及依法不能建築之土地** |
| 級 1 | 相鄰街廓同歸戶合併 |
| 級 2 | 鄰近街廓同歸戶合併 |
| 級 3 | 非鄰近街廓同歸戶合併 |

例（弱弱聯合）：投影序 5,3,6,4,1,8；3、4、1 同歸戶皆不達標，3 的 G 最大
→ 4、1 併入 3 → G(3+4+1) 達標 → 以 3 的投影位次分配。

4 級走完仍無可合併者 → 定義為「**無法分配亦無同歸戶的 R**」，進七-4。

### 7-2. 第 2 調配順位：RD 道路用地（手冊 192-193 頁）

先依手冊道路用地分配原則做同歸戶合併（無視調配池事前檢查）：

1. 兩側皆有同歸戶土地 → 以 CENTERLINE 切兩塊，各自**整筆**分向兩側合併。
2. 一側合併後未達最小分配面積 → 向另一側集中。
3. 一側屬公設地 → 向可建築側集中。

觸發轉出條件：「兩側街廓原有土地分配後已無空地（= 調配池已被第 1 順位用完）
可供分配道路用地，或兩側本無土地可供合併」
→ 定義為「**無法分配亦無同歸戶的 RD**」，進七-4。
此類土地以**重劃前整筆**面積調配（手冊規則 4，需合議制；試分配以七-4 預設自動跑）。

### 7-3. 第 3 調配順位：共同負擔公設地（手冊 194-195 頁）

廣場、鄰里公園、市場用地等。相鄰住宅區有同歸戶他筆土地 → 整筆合併分配
（無視調配池事前檢查）。但若**整筆合併會超過該街廓剩餘調配池** → 不能硬併，
轉出定義為「**無法分配亦無同歸戶的 PF**」，進七-4（需合議制；試分配預設自動跑）。

### 7-4. 三類需調配土地的歸戶合併調配（無同歸戶 3 級調配）

**前置**：將同一歸戶的 R + RD + PF 全部加總：
- 合併重劃前總面積 = Σ各筆面積
- 合併重劃前總地價 = Σ(各筆重劃前地價 × 面積)（不同位置地價不同，涉價值換算）
- **暫定位置錨點** = 重劃前面積相對最大的那筆土地位置（供 3 級調配算距離）

**排序**：多個歸戶搶同一最近街廓時，重劃前總地價高者先調配。

**3 級順序**：相鄰 → 鄰近 → 非鄰近（自暫定錨點起算距離），且遵守
「最近優先；最近街廓調配池不夠或會使該地不達最小建築面積 → 次近或門檻較低街廓」。

**a' 折算雙模式**：
- 歸戶在目標街廓有重劃前土地：a' = a × p(原位置) / p(目標街廓該歸戶地)
- 無：a' = a × p(原位置) / p_average(目標街廓)（DXF 幾何面積權重平均地價）

**同級多歸戶比例分配**：分母綁 level、逐層消耗 capacity、每層分母獨立；
灌入上限 min(G'(g), cap_share)；G 值層級與重劃前面積層級透過 G 公式迭代收斂，
**收斂門檻 0.05㎡**。同級皆無同歸戶（戊己情境）→ 平均分配、多輪平均。

**調配池消耗規則（不留中間態）**：
```
a' 量 ≥ 池可吃量 → 池吃光歸零，剩餘 a' 轉下一街廓
a' 量灌入後池仍 ≥ 該街廓最小分配面積 → 正常灌
a' 量灌入後池會落入 (0, 最小分配面積) 中間態 → 只灌到池=最小分配面積，剩餘轉走
```
且調配進去的土地本身經 G 值計算須 ≥ 目標街廓最小建築面積（概念 4）。

**最佳化調配（三條件交集）**：
歸戶合併總面積**不必整筆**塞同一街廓——可拆部分面積調配，目標是同時滿足：
3 級調配距離優先序 ∩ 調配後該地達目標街廓最小建築面積 ∩ 剩餘調配池不落中間態。
程式以試算模擬求最佳化解。

### 7-5. 1/2 門檻雙路徑（手冊宗地分配原則(四)(五)）

歸戶合併後總應分配面積與「重劃區內最小分配面積標準」比較：

**≥ 1/2**（原則(四)）：
得於**深度較淺或重劃後地價較低之街廓**按**最小分配面積標準**分配
→ 實際分配面積 > 應分配面積 → 向該歸戶**收取差額地價找補**。
KL 政策：**盡量留設「重劃區內最小分配面積那個街廓」的調配池**，
專供此類 1/2 以上歸戶配置。

**< 1/2**（原則(五)），三條路：
1. 申請與他人合併分配為**持分共有**（選填介面，見 7-6）
2. **單獨調配**：於深度較淺、地價較低之街廓若達該街廓最小分配面積 → 該街廓計算分配
3. **合併調配**
試分配預設：程式先自動試「單獨調配」→ 不行才標記進選填介面。
皆不成 / 不同意合併 → **現金補償**。
現金補償判準：無同歸戶 且 G 值 < min(街廓最小分配面積) × 1/2。

### 7-6. 選填介面（正式作業用；試分配顯示預設值）

需合議制調配的土地清單：地號、面積、自動預設去向（街廓+調配池）、
可下拉覆寫「實際調配去向」。
持分共有選填：地主選目標街廓之重劃後地號（可選多個）；
若所選街廓全比調配後剩餘調配池 < 最小面積或不夠 → 程式自動請地主選填第 2 順位；
不同意合併分配者 → 領現金。
