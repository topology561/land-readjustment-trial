## 1. 鎖定定義（KL ground truth，取代舊口語「五邊形」）

### 1.1 街角最小分配面積範圍

由下列邊圍成的**多邊形**（矩形街廓退化為五邊形，一般街廓邊數不固定）：

```
FRONT_LINE ＋ 道路截角斜邊 ＋ SIDE_LINE ＋ BASELINE ＋ ALLOC_LINE(@ W_i = T)
```

- **T（框寬）= 街角退縮（使用者輸入，case/block 級參數）＋ 該街廓臨正街寬對應之最小畸零地寬（查花蓮畸零地表）**。
  手冊圖 7 例：T = 3.5（退縮）＋ 5（最小建築寬）= 8.5 m。
- **街角地 W_i**：取 SIDE_LINE 上「與道路截角斜邊之交點」到「與 BASELINE 之交點」的子線段，其**中點 M**；W_i = M 到「本筆與下一筆之 ALLOC_LINE」之 **⊥ALLOC 垂距**。
- 把該 ALLOC_LINE 擺到 **W_i = T** 的位置 → 它即為第 1 宗街角評比框的內邊。
- 此框由 T 純查表導出，**與 §3 推進無關**，無 D-1 之 w_i↔k 循環依賴。

> 現況：`_build_corner_range_v2`(app.py 7410) 已實作此框——`shift_distance = _setback + 法定最小寬 = T`(7823)、沿 ALLOC 法向平移、切真實 BLOCK 取角側片、`difference` 截角三角形(7450-7454)、回傳一般多邊形（非寫死五邊）。**此函式邏輯正確，本波不重寫，僅修正其截角來源（見 §4）與精度項（見 §5）。**

### 1.2 正街角線（KL 更正）

**正街角線 = 道路截角的斜邊（視距截角 / cut corner for sight line），非 FRONT_LINE。**
臨正街角線長 = 宗地邊界落在該截角斜邊上的長度。直接面對街角截角斜邊才是街角地溢價來源；單純臨正街（FRONT_LINE）不計入項一。

### 1.3 §1 三指數（手冊圖 8）

| 指數 | 權重 | 分子（逐筆） | 分母（範圍自身） |
|---|---|---|---|
| 項一 臨正街角線 | 0.4 | 該筆 parcel∩range 邊界落在**截角斜邊**上的長 | 截角斜邊總長 |
| 項二 臨側街線 | 0.2 | 該筆 parcel∩range 邊界落在 **SIDE_LINE** 上的長 | range 之 SIDE_LINE 邊總長 |
| 項三 跨占面積 | 0.4 | `_corner_intersection_area`（= 該筆 parcel∩range 面積） | range 面積 |

三項皆為「**各筆 vs 同一個範圍**」的逐筆量，且皆可由「parcel∩range 那塊多邊形的邊與面積」一次拆出。

### 1.4 第 1 宗街角地為抵費地之情形（手冊圖 9，KL 提醒）

當該街角各候選重劃後配回 G **皆未達**街角最小分配面積（全數未過 §6 G 門檻）時，第 1 宗街角地**留設為抵費地**（forced offset），其**面積 = 「街角最小分配面積範圍」（§1.1 之 T-多邊形面積），不多不少**。

關鍵守恆（**與 §3 W-D.2 滑池槽耦合**）：
- 該街廓**中央仍照常產生調配池**——抵費地占角 ≠ 取消中央池。
- 角落抵費地之面積是「**該街廓總池（抵費地）的一部分被重新定位到角落**」，非額外新增面積。
- 故守恆為：`街廓面積 = Σ(私有 G) + 角落抵費地(=range面積) + 中央調配池`，其中 `角落抵費地 + 中央調配池 = Stage 1 產生之總池`。§3 計算中央池時須扣除已定位到角落者（`中央池 = 總池 − 角落抵費地`），守恆 <1㎡ 不變。

§1 之 PK 仍照常評分以判定是否有合格 winner；全數不合格 → 抵費地。此為**合法輸出（非失敗）**。現碼以 `forced_offset`/`強制抵費地`/`f3L_forced_offset`(7560-7563,14104-14117) 表達；本波須保留此路徑，並確保抵費地面積鎖定為 range 面積、且把「角落抵費地屬重新定位之池面積」傳遞給 §3/Phase C 守恆（避免被當成額外面積而破守恆）。

## 4. 截角辨識改「拓樸定錨」（KL 裁示 = Option A，不新增圖層）

**規則**：截角斜邊 = 在 BLOCK 多邊形上，夾在「FRONT_LINE-吻合之 BLOCK 邊」與「該側 SIDE_LINE-吻合之 BLOCK 邊」之間的那條 BLOCK 邊。

- FRONT_LINE 與 SIDE_LINE(left/right) 對 BLOCK 邊的吻合，採**共線＋投影重疊**（見 §4.1；**非**端點重合——實資料已證端點法六塊全滅）。
- 每側截角 = 連接 FRONT-吻合邊 與 該側 SIDE-吻合邊 的 BLOCK 邊：
  - 恰 1 條 → 該側截角斜邊（取其兩端點）。
  - 0 條（FRONT 與 SIDE 共頂點）→ 該側無截角，項一分子=0、分母防呆。
  - ≥2 條 → 多段截角，warn 並交 KL 判（暫取合併投影或拒絕）。
- **廢除長度啟發法**：`classify_block_edges` 之 `cutoff_idxs` 長度區間判定(4466-4467)整段刪；下游 `theoretical_corners`/`_make_chamfer_tri_wb`/`_build_corner_range_v2` 一律改吃拓樸定錨之截角邊。保留 `cutoff_idx` 對外介面欄位名以減少下游波及（由 CC＋reviewer 評估介面相容做法）。
- 截角斜邊為**單一真相源**：同時供（a）`_build_corner_range_v2` 之截角扣除、（b）項一之正街角線長。BLOCK 仍為唯一幾何來源，零 sliver 風險。

### 4.1 -a 配對修正（實座標驗證，claude.ai 用 KL 提供之 SVMAP 座標檔對位）

**根因（實資料證實）**：CC 的 `_anchor_chamfers_topology` 以「端點重合(0.5m)」配 FRONT/SIDE→BLOCK 邊，退縮 0m/3.5m **六塊全滅**（新 front_idx 空、新截角 []、左右皆「配對失敗」）。原因不是長度法測錯，而是：CAD 的 FRONT_LINE/SIDE_LINE 是**畫到理論(未截角)街角的長線**（本案 FRONT 段 85–100m、SIDE 33–45m），而 BLOCK 的正街/側街邊**停在截角處**、比線短約 3.5m（截角腿長）。BLOCK 邊精確躺在線上（垂距≈0、共線），只是線端外伸 ~3.5m → 端點法一端合、一端差 3.5m → 全塊皆敗。

**修正規則（通用、零案例常數）**：FRONT/SIDE→BLOCK 邊改「**共線＋投影重疊**」——
- 方位角 mod 180 相差 < ~2°；
- BLOCK 邊兩端點到該線（無限延伸）垂距 < ~1m；
- BLOCK 邊中點投影落在線段（或聚合線）span 內。
- 實測 BLOCK 邊到線垂距≈0、方位差≈0，故 2°/1m 僅為捨入緩衝、**非為湊本案調的擬合參數**。此規則對六塊之正街/側街邊全數乾淨配對。
- 截角仍為「拓樸夾在 FRONT-配對邊 與 SIDE-配對邊 之間的 BLOCK 邊」（Option A 不變），配對來源換共線即成立。

**為何 FRONT/SIDE 線畫到未截角街角（KL 域理，務必保留；勿把線改成停在截角）**：
- G 公式的 **F（＝街角地第 1 筆面臨側面道路之長度）＝ SIDE_LINE 全長**（畫到未截角角）。
- 臨街地特別負擔總面積：**正面道路負擔總面積 ＝ FRONTLINE 長 × 正面道路負擔尺度**（按面臨正街寬查表）；**側面道路負擔總面積 ＝ SIDE_LINE 長 × 側面道路負擔尺度**（按面臨側街寬查表；側街取半見已鎖 l1 定義）。
- 這些量都依賴「未截角全長」，故 FRONT/SIDE 線**必須**畫到理論尖角。截角只在「街角範圍多邊形」與「項一正街角線」用到，不影響 F／負擔總面積的長度計算。
