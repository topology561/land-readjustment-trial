# W-D.4 合併分配規格（分級清單波，claude.ai 撰）

> 性質：**施工規格**。CC 據此進 plan mode → reviewer → harness 綠 → push。
> 波性質：**清單波**。零幾何移動、零財務實算。動的只有：新診斷輸出、harness 閘、文件。
> 停機條件：任何 G／守恆／位次／J／k* 逐格變動＝越界。run_all 既有全部閘全綠＝零移動之證。
> 前置：《W-D.4_域裁鎖定.md》兩條 backlog（ALLOC 靜默、硬編住宅區）**先清再開工**。

---

## 1. 目標

把 22 筆 `⚠️移出/第1調配順位` 旗標（兩情境同名單）**消費**成一份歸戶級四梯分級清單：每一歸戶標明所處梯次、對應 v3 級別、建議路徑與參考估算值。此清單為 W-F 七級調配引擎之**驗收準源**。

## 2. 輸入

| 來源 | 內容 |
|---|---|
| G 值計算結果（v2 baseline 同源管線） | 逐宗 G、S、宗地寬度、畸零地旗標、所屬街廓、原地號、重劃前區段 |
| Tab1 清冊（快照同源） | 所有權人歸戶鍵、持分、**他項權利欄、禁止移轉欄**（本案全空） |
| 參數表（快照） | 逐街廓：分配深度、分區、正面路寬 |
| 畸零地附表查表 | min_width_i = f(分區_i, 正面路寬_i)（**逐塊查，禁寫死**） |
| Tab4（快照） | 逐街廓重劃後評定地價（R1/R4 99,424.43；R2/R6 94,453.21；R3/R5 89,481.99） |

**術語（KL 釘定）**：應分配權利面積＝G；重劃前原有面積＝a。法源：辦法 §31（分配方法）、§52（差額地價）、§53（現金補償兩式＋放棄改領）、§26/27（臨街地特別負擔尺度，引擎已全表實作）。

推導量（**harness 內算並斷言，禁寫死**）：
- 逐街廓最小分配面積 `MinA_i = 分配深度_i × min_width_i`
- 區標準 `MinA_區 = MIN_i(MinA_i)`；1/2 線 `= MinA_區 / 2`
- UC9898 期望值（斷言錨）：MinA_區 = 114.07（R4）、1/2 線 = 57.03

## 3. 演算法（四步）

### 步0・歸戶聚合
- 以所有權人鍵 group 全部業主宗地（不限 22 筆——梯次判定以**歸戶總額**為準，達標戶也要能證明其達標）。
- 歸戶應分配面積 `ΣG_戶 = Σ G_i`（各宗於原街廓計算之 G 直加；**不做 a′ 換算**——a′ 僅於實際跨街廓改配時生效，屬 v3）。
- 共有宗地：依持分拆算至各共有人歸戶（持分×G）。持分拆併之進階機制（v3 附B 待議項）本波不碰，僅拆算計入。

### 步0.5・第4梯前置過濾
- 歸戶內任一宗他項權利欄或禁止移轉欄非空 → 該歸戶標 `阻卻=是`（不得合併分配；仍可原位分配）。
- 本案預期全空 → 全數 `阻卻=否`，此為有效驗收結果。
- **宣告紀律**：欄位缺失（非空值、是整欄不存在）→ RuntimeError，不得靜默視為無阻卻。

### 步1・逐宗達標判定
- 宗 i 達標 ⟺ `G_i ≥ MinA_(所屬街廓)` **且** 該宗無畸零旗標（S=0 或寬<最小寬者即使 G 夠大亦不達標——裁示 1 幾何準則優先）。
- 歸戶內全數達標 → **第1梯**：逐宗個別分配，無動作。

### 步1.5・同宗跨占分配線判定（辦法 §31-1-3）
- 同一原地號被 overlay 拆為多街廓暫編（如 628-1(1)/(2)）者：各側 G 對**該側原街廓 MinA** 個別判；
  未達側標 `跨線併大側` 並指向面積較大側之暫編（清單標註，搬移屬 W-F）。
- 此判定先於歸戶聚合之梯次分流（宗級規則優先於歸戶級）。

### 步2・歸戶級分流
對含未達標宗之歸戶，依 `ΣG_戶`：

| 條件 | 梯次 | 路徑標註 | 參考估算（標「參考值，a′ 待 v3」） |
|---|---|---|---|
| ΣG_戶 ≥ MinA_(某可承接街廓)，即集中後可達標 | **第0梯** | 集中合併於應分配面積較大之宗（列標的宗暫編地號）；不得合併於公設地及依法不能建築地 | — |
| MinA_區/2 ≤ ΣG_戶 且集中仍無法達任一街廓標準 | **第2梯（雙出口）** | 出口a：協調共有（預設不成）→ 增配（§31-1-2 後段＋§52-1）。候選目標：最淺深度街廓（R4）＋最低後地價街廓（R3/R5）。出口b：按標準配後**申請放棄改領現金**（§53-2） | 出口a：增配面積≈MinA_目標 − ΣG_戶；差額地價≈增配面積×目標街廓後地價。出口b：放棄補償≈ΣG_戶×分配位置後地價 |
| ΣG_戶 < MinA_區/2 | **第3梯（兩式）** | 申請與他人合併共有（預設不成）→ 現金補償不配地（§31-1-2 前段＋§53-1） | 本文式：補償≈Σa_i×原位置後地價；但書式（勘定後自行分割、繼承/強執除外）：Σa_i×原位置**前**地價（旗標欄，前地價 v2 半中間態下僅標式不算值）；§53-3 他項權利協調旗標 |

（第0梯標的宗選擇：歸戶內 G 最大之達標宗；若歸戶內無達標宗但 ΣG 集中後 ≥ 該宗所在街廓 MinA，仍屬第0梯集中——即 v3 級0' 弱弱聯合。）

### 步3・S=0 抵費地遞補標註（不執行、只標註）
- 對 W-D.3 三分類之每一「碎片」，依域裁 (B) 搜尋規則沿分配序往前找**遞補標的宗**（第一筆 S≠0 且宗地寬度≥min_width_i 者，跳過失格筆），輸出 `碎片→標的宗` 對照（UC9898 斷言錨：R6 85.66 → 628-1(2)）。
- 同角落有失格業主者，加註 `定序=先解業主`（域裁 C）。
- 形狀調整本體屬 W-F，本波僅產標註。

## 5. 驗收閘（入 run_all）

1. **零幾何移動**：既有全部閘（18＋W-D.3 三閘）對凍結 baseline 逐格不變。
2. **旗標全消費**：22 筆畸零旗標宗，每筆恰出現在某歸戶列之宗清單（無遺漏、無重複）。
3. **推導斷言**：MinA_區==114.07、1/2線==57.03 由參數推導而得（改參數表任一分配深度，值須隨動——reviewer 反向篡改測試）。
4. **遞補標註錨**：R6 85.66 → 628-1(2)、跳過 628(2)。
5. **第4梯 fixture**：harness 合成 1 筆帶查封註記虛構宗（不入 baseline、不碰 Tab1），斷言其歸戶 `阻卻=是` 且被排除於第0梯標的宗候選；移除 fixture 後輸出與 baseline byte-identical。
6. **決定性**：兩次乾淨執行 byte-identical（`lineterminator='\n'`）。
7. 清單 CSV 轉正 v2 baseline＋PROVENANCE 補記（產出 commit、參數身分、「參考估算非實算」聲明）。

## 6. 邊界與禁區

- 不動 app.py 分配引擎；app 側至多加一個唯讀清單展示（可選，非本波必要）。
- 不呼叫 `_allocate_three_tier_v1` 等三個待廢函式。
- 不算 a′、不做增配/補償/共有拆併實算、不跨街廓搬地。
- 歸戶鍵、他項權利欄名以 Tab1 實際欄名為準（CC 偵察後入 plan，禁假設）。

## 7. 與後續波之接線

- 本清單＝W-F 七級引擎驗收準源（每歸戶「應走路徑」為 W-F 之 expected）。
- v3 財務接線後：參考估算欄升級為實算（a′、增配、差額地價、補償），本規格第 3 節步2 之公式即實算規格。
