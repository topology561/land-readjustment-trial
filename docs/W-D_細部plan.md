## §0. 波次邊界（先鎖死，避免把後波的事偷做進來）
W-D **只做位次決定**，以下一律**不做**、維持 W-C 收斂後的狀態：
- **不灌 a'**：`面積_m2` 全程維持 0（W-C 已鎖）。位次決定不改變 a 的組成（a=分攤登記面積_m2）。
- **不跨街廓搬地**：畸零地移到別塊、同歸戶跨塊合併 → **W-E**。W-D 只在**同一街廓內**定序與滑池槽。
- **不做公設地補償**：Tier 1 `_TIER1_APRIME_INJECT_ENABLED` 維持 **False**（W-F 才翻 True）。
- **守恆不變量**：每塊 `ΣG_lot + Σ抵費地 = 街廓 DXF 面積`（差 <1㎡）全程必須成立。位次重排只在「ΣG ↔ 池」與「各筆之間」移動幾何，總量不變。

## §1. 街角第 1 筆選定（街角評點）
**法源（市地重劃手冊 街角地分配原則）**：街角地第 1 宗應為**重劃前坐落於街角最小分配面積範圍內、且計算分配結果能達街角地最小分配面積標準**者。數宗競爭時，以**跨占該街角規定範圍內臨街面較寬、跨占面積較大者優先**，並依下列指數定序：

| 指標 | 權重 |
|---|---|
| 臨正街角線（占最小寬之比） | 0.4 |
| 臨側街線（占側街範圍之比） | 0.2 |
| 跨占街角規定範圍內面積（占範圍面積之比） | 0.4 |

**實作要點**：
1. **街角範圍多邊形**用 `_build_corner_range_v2`（W-C 鎖定：不得用矩形近似 `深度×最小寬−截角`）。最小分配面積＝該範圍多邊形面積（含截角扣除）。
2. **候選**＝重劃前宗地與街角範圍多邊形有交集者；逐筆算三項指數、加權求總分。
3. **資格門檻**：候選分配後須達街角地最小分配面積標準，否則不得排第 1 宗。
4. **每個有臨側街的街角各自選 1 筆**（街廓兩端各有街角 → 兩個第 1 宗，左右各一）。無側街臨路的端點不選街角、以 BLOCK 中點為 W 起算（W-C plan §1）。
5. 評點過程與分數**輸出診斷表**（街廓／候選地號／三項分數／總分／是否達標／選中與否），供 reviewer 與 KL 核。

## §2. 其餘各筆位次：原位次 = 重劃前投影順序
- **原位次定義（鎖定）**＝重劃前各宗地在該街廓內、沿 ALLOC 法向（臨街向）的**投影順序**。例：R2 左到右 = 1,2,3,4,5。
- 街角第 1 宗依 §1 選定後，其餘筆**依投影順序**自兩端向中央排列（**兩端往中間夾**）。
- **調配池不是位次**：池不占編號，其槽位可落在序列任一處，故 `1,池,2,3,4,5`、`1,2,3,4,池,5` 在原位次上都等同 `1,2,3,4,5`。池槽由 §3 決定。

## §3.〔本波頭號機制〕調配池槽位 + 側街負擔最大化（跨側 Rw）
**目標（鎖定）**：選調配池槽位，使**兩側私有地實扛的側街負擔總和最大**：

`目標 = Σ_兩側 [ min(R(該側末筆W), 100%) × F_側 × l₁_側 ]` → 取最大

- 約束：保持原位次投影順序（§2）不動，**池槽為唯一自由變數**；候選槽位 ∈ {序列各間隙}。
- 「兩側各達 100%」只是 `ΣW_私有 ≥ 36m 且離散地號切得開`時的最佳特例，**不是目標本身**。目標是上式的最大化。

**為何是「最大化」而非「平衡到 100%」（凹函數 + 飽和）**：
- Rw 表是**凹函數、W=18m 飽和**（R(18)=100%，之後封頂）。把寬度堆在**已過 18m 的飽和側**，超出部分**邊際 Rw=0、純浪費**；挪到**還在陡升段（W<18m）的不足側**才有淨增益。
- 實例（兩側等權）：`1,2,池,3`（w=12,24,6）左 R(36)=100%＋右 R(6)=50% = **150%**；改 `1,池,2,3` 左 R(12)=80%＋右 R(30)=100% = **180%**。守原位次、池仍中段，多收 30%。

**F×l₁ 加權（一般化）**：兩端側街 `F×l₁` 可能不同、每 1% Rw 換算的負擔**面積**不同，故目標以**面積**加權（上式）。兩側等權時退化為「加 %」比較；不等權時優先餵滿 `F×l₁` 較大側。
- **F_側 = 街角第 1 筆面臨側街之長度 ＝ 該側 SIDE_LINE 長度**（平行 ALLOC 條狀模型中，街角第 1 筆沿側街那條邊＝該筆全深＝整條 SIDE_LINE）。
- **l₁_側 = 該側街路寬查臨街地特別負擔尺度表**（側街取正面標準之半，第 26 條：<4m→0、4–8m→(路寬−4)/2、8–20m→路寬/4、≥20m→5，再 ÷2）。

**「居中」是副產品**：Rw 凹、偏好平衡 → 最大化會自然把池往中央拉；池居中多為**結果**、非獨立目標。離散地號決定實際可落槽；少數兩側 `F×l₁` 差大時，**最大化勝過幾何置中**。

**法理 + 池 reserve**：
- 臨街地特別負擔由**臨街私有地**按各自 W 深度（Rw）自 G 扣除；該負擔之地成為**道路（街廓外）**，不屬本街廓調配池。
- **max ΣRw ⟺ max 該塊 stage-1 調配池**：G 公式 Rw 帶負號（Rw 大 → G 小 → 池大）。故本節「最大化側街負擔」同時就是「**最大化該塊池 reserve**」，供後續波次調配。
- **B、C 為全區常數、不受逐塊 ΣRw 影響**：由全區 gross 算定、固定；某塊 ΣRw 未達 100% 不回頭改 B/C。
- **調配池為未分化殘餘（≈ 公設地平均負擔 B + 費用平均負擔 C）**：ΣRw 偏低 → ΣG 偏大 → **池偏小（池「吸收」差額）**；池變小**無法區分**來自 B 部分或 C(抵費地) 部分。（先前「抵費地不扛特別負擔」之說係全區總額計算用語、誤用於此，已移除。）
- **填不滿 → 該側 ΣRw<100% 為合法**：該塊 stage-1 池較滿載時小；缺口由**後續波次（合併/公設地）自中央池『較低 Rw 側』漸次消化**補足至 100%（原則上，見 §5）。**不強制湊私有**。
- **逐街廓獨立最大化（鎖定）**：每塊各自最大化（不做全區聯合最佳化）。理由：(a) 每塊先得最大池 reserve；(b) 各塊原則上後續補滿至 100%；(c) 各塊位置有好壞，獨立最大化保留最多後續調配選擇。B/C 不動，無需全區校驗。

**適用範圍（先分流，池策略不同；但「居中為預設」對三者皆適用）**：
- **雙側臨路**（兩端各有側街）：本節滑池槽最佳化完整適用（上式兩項、跨側權衡）；居中為底，按 Rw 微調。
- **單側臨路**（僅一端側街）：只有一項 Rw。**池仍以居中為預設**；僅當該側 Rw<100% 時，把池往**遠離側街方向**移（讓負擔側多納幾筆）以補到 100%，**達標即止、不過度外推**（例：若第 1 筆即達 100%，池維持居中 `1,2,池,3,4`，不推到 `1,2,3,4,池`）。
- **無側臨路**（僅正面道路）：Rw 全 0；池居中，位由正面負擔 + §4 碎片合併決定，無本節最佳化。
- **通則**：居中為預設；Rw 已飽和（池位不影響 Rw）時回居中；唯 Rw 仍可提升時才偏移，且偏移取「達成 max ΣRw 的最小偏離中央」。

**機制（滑池槽，雙側臨路）**：
1. **滑池槽 = 決定左／右群分界**：池槽左側各筆從左街角累積 W、右側各筆從右街角累積 W。槽位每移一格，某筆改由另一街角餵、其 Rw 隨之變。
2. 對每個候選槽位，**呼叫 W-C 數學引擎**算兩側 `ΣRw_側 = R(末筆W) − R(起始W)`（telescoping）與上式加權目標值。
3. **選槽準則**：取目標值最大者；平手（或差異 <ε）時取**池較中央/碎片較少**者（接 §4）。
4. **填不滿**：若該街廓私有地總量本就填不滿、無溢出側可餵，標記該側「`ΣRw<100%（私有不足，差額由該塊池吸收、B/C 不變，待後續波次消化補滿）`」，不做人為調整。
5. **可補滿訊號**：某側末筆 W 遠超 18m（溢出）才有餘地餵不足側。例：R1 左末筆 42.7m、R4 左 46.87m 大幅溢出 → 對側很可能可補。

> 註：W 兩個量別混（W-C plan §1）——判去留用單筆 `w_i`、本節 Rw 用累積 `W_i`。

### §3.1 選槽演算法（Claude Code 可直接照做）
雙側／單側／無側**共用同一套**；分流只靠「該端 side 是否 has」決定該側項是否為 0。釘選的街角地不可被池頂掉。

```
輸入（逐街廓）:
  P = [p_1..p_n]            # 原位次（投影序，左→右）；p_i.w = 單筆宗地寬度 w_i（⊥ALLOC, W-C §1）
  sides.left  = {has, F_left,  l1_left,  b_left}   # has: 該端是否臨側街
  sides.right = {has, F_right, l1_right, b_right}  # F = 該側 SIDE_LINE 長；l1 = 路寬查負擔尺度表(側街取半)；b = forced 起始W(無則0)
  R(W)                      # 法定 Rw 累積函數：R(0)=0 … R(18)=100%，W>18 → 100%
  pinned: 若 left.has 釘 p_1 於左角；若 right.has 釘 p_n 於右角（§1 街角評點選定）

STEP 1  候選槽 K（slot k：左群={p_1..p_k}、右群={p_{k+1}..p_n}，pool 插中間）
  k_min = 1   if left.has  else 0     # 有左街角 → 左群至少含 p_1（不可頂掉釘選）
  k_max = n-1 if right.has else n      # 有右街角 → 右群至少含 p_n
  K = [k_min .. k_max]
  # 推得：雙側 K=[1..n-1]；單側左 K=[1..n]；單側右 K=[0..n-1]；無側 K=[0..n]

STEP 2  逐槽算加權目標 J(k) 與偏離 dev(k)
  for k in K:
    L_far  = b_left  + Σ_{i=1..k}   w_i          # 左群末筆累積 W（含 buffer 起點）
    Rt_far = b_right + Σ_{i=k+1..n} w_i           # 右群末筆累積 W（自右街角）
    ΣRw_L  = (R(L_far)  - R(b_left))  if left.has  else 0
    ΣRw_R  = (R(Rt_far) - R(b_right)) if right.has else 0
    J(k)   = ΣRw_L*F_left*l1_left + ΣRw_R*F_right*l1_right       # 加權「負擔面積」, 取最大
    dev(k) = | Σ_{1..k} w_i  -  Σ_{k+1..n} w_i |                  # 私有左右寬差 = 偏離中央程度

STEP 3  選槽（最大化為主、居中為 tie-break）
  J*     = max_k J(k)
  cand   = { k in K : J(k) >= J* - ε }     # ε 取極小(僅吃浮點/飽和平坦段的真平手, 例 1e-6*J*)；不為居中犧牲實質負擔
  k_pick = argmin_{k in cand} dev(k)        # 平手再取較小 k（序列穩定可重現）

STEP 4  落位
  序列 = [p_1..p_{k_pick}] ++ [POOL] ++ [p_{k_pick+1}..p_n]     # 原位次不動、僅插 pool
  輸出診斷: 各 k 的 J/dev、選中 k、ΣRw_L、ΣRw_R（供 reviewer/KL 核）

邊界情形（皆由上式自然涵蓋、無需特判）:
  E1 無側街        : J(k)≡0 → cand=K → 取 dev 最小（池居中）
  E2 單側·首筆即飽和: J 在 k≥k0 平坦(=100%) → cand=平坦段 → dev 最小（池回居中, 如 1,2,池,3,4）
  E3 單側·永不飽和  : J 隨 k 嚴格遞增 → k_pick=k_max（私有全擺負擔側, 池貼遠端, 如 1,2,3,4,池）
  E4 forced buffer : ΣRw 自 R(b) 起算（已含於 STEP 2）；buffer 段不灌私有 Rw
  E5 單筆過寬越18m  : 超出段邊際 Rw=0（R 飽和自動處理）
  E6 §1 角地非投影端: 以 §1 釘選放角、其餘按投影序；§3 take pinned as given
  E7 兩側皆 <100%   : J 仍取最大（盡量收）、標記「未滿待後續波次消化池補滿」（§5）

約束: 不改 a(面積_m2=0)、不跨塊搬地、Tier1 gate=False（§0）；落位後 ΣG_lot+池=街廓DXF面積(<1㎡) 守恆仍須成立。
```
> 驗證對齊：n=3, w=[12,24,6], 雙側等權 → J(k=0)=100、J(1)=180、J(2)=150、J(3)=100 → k_pick=1（`1,池,2,3`，180%），與 KL 例一致。

## §6. 驗收清單
1. **街角第 1 宗**：每端街角由評點選出、達街角地最小分配面積標準；診斷表三項分數可複核。
2. **位次**：其餘筆 = 重劃前投影順序、兩端往中間；池不占編號。
3. **池槽 + 側街負擔最大化**：池槽取「兩側私有實扛側街負擔加權總和最大」者；輸出各候選槽的目標值供複核；可補滿側補至 ~100%（如 R1右/R4右）、填不滿側標記「差額由該塊池吸收、待後續波次消化補滿」。
4. **碎片分類（W-D.3 診斷；裁示1＋Q2）**：分配後每筆土地**含調配池**須：
   (a) **宗地寬度(⊥ALLOC) ≥ 法定最小寬**、(b) 封閉多邊形 **≥1 邊與 FRONT_LINE 重疊（臨正街）**；
   最小深度(∥ALLOC)≥法定最小深 亦列（BASELINE 使用者繪、劃設時通常已符，預期恆過）。
   **驗收分流（Q2）**：非-forced 街廓＝池主體單一連通片＋允許已標籤碎片（各附「不可連通理由」＝
   逐邊分類）；forced 街廓＝角落鎖定地＋中央池各自連通＋碎片同上。
   **S=0 判準**：臨正街長=0 ⟺ 不臨 FRONT_LINE ⟺ S=0（與逐邊分類同一把尺）。
   碎片之實際合併/併池屬 §8 W-D.4 行為波。
5. **守恆**：每塊 `ΣG_lot + 池 = 街廓 DXF 面積`，差 <1㎡。
6. **波次邊界未越界**：`面積_m2=0`（未灌 a'）、Tier1 gate 仍 False、無跨塊搬地；
   **W-D.3 診斷波另加：零幾何移動**（run_all 既有閘全綠＝G/守恆/位次/J/k* 逐格未動之證）。
7. **不被波及**：W-C 已收斂項（深度、軸向 ⊥FRONT、白縫 gap≈0）不受位次重排影響。

### §6-A 兩欄語意 doctrine（W-D.3 釘定，KL 確認）
- **`W(m)`＝側街負擔寬**（隨 l₁/Rw 沿側街累積）；某側 l₁=0（無臨側街／臨已開闢公有路）時
  該推進方向 `W(m)=0` **為正確、非 bug**（如 R6 左推全序列 W(m)=0）。
