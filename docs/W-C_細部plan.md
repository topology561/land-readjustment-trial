### W 的法定定義（附件二·符號 W）——**這是「W 方向基準改 f3_cad_alloc_dir」的法源**

> 「W 表示分配土地寬度（**宗地側街臨街線實際長度之中點，向宗地分配線作垂直線**所量其間之距離）。」

拆解：
- 基準點 = 宗地側街臨街線之中點 = **SIDE_LINE 中點**。
- 量測方向 = **向宗地分配線（ALLOC_LINE）作垂直線** = 沿 ALLOC_LINE 的**法向**量距離。
- 因各筆地界線都平行 ALLOC_LINE，「垂直 ALLOC_LINE 的垂距」即「沿 ALLOC_LINE 法向量的距離」。

→ 故 W **必須**以 `f3_cad_alloc_dir`（W-B §1-3 讀入的宗地分配線方向）的法向為基準。
**用 MBR 較長邊當方向是違反法規定義的**（斜街廓 MBR 較長邊 ≠ ALLOC_LINE 法向 → W 量錯 → Rw 查錯 → 畸零地判錯）。鐵律 G/H + 附件二三者同指此修正。

### 🔒 兩個量：單筆宗地寬度（increment）vs W累積（cumulative）——KL 鎖定，勿混
方向都是 ⊥ALLOC（`n_alloc = rot90(f3_cad_alloc_dir)`）；差別在「單筆增量」還是「從側街起算的累加」：

- **單筆宗地寬度 `w_i`（increment）** = 本筆兩條邊界 ALLOC 線的 ⊥ALLOC 間距。
  - **第1筆（街角地）**：`SIDE_LINE 中點`（側邊無臨路則 `BLOCK 中點`）⊥ 到第1條 ALLOC 線的距離。
  - **第2筆起**：各 ALLOC 線**彼此平行** → `w_i` = 相鄰兩 ALLOC 線的**平移距離**（平行線間距，量任一點皆同，「中點」在此不影響）。
  - **用途**：**畸零地判去留**（`w_i < 最小寬 → 移出/第1調配順位`），逐筆、各自獨立。
- **W累積 `W_i`（cumulative）** = `Σ_{k≤i} w_k`（從起始邊界 SIDE_LINE 一路加總到本筆遠側）。
  - **用途**：**僅給 Rw**。`Rw(第i筆) = R(W_i) − R(W_{i−1})`（差額）；`ΣRw_側 = R(末筆W) − R(起始W)`（telescoping，§3）。
- **務必區分**：判去留用 `w_i`、Rw 用 `W_i`。實證 628-42(2)：`w=1.92`（<3.5 → 旗標）、`W累積=6.45`（給 Rw 差額）——兩者不可互代。
- code 對應：`_mark_zaling` 的 `宗地寬度 = S·|d_hat·n_alloc|` 幾何上＝相鄰兩平行 ALLOC 線 ⊥ 間距＝`w_i`（規格與程式吻合）。
- 📐 註（弧形/斜側街精度，本案矩形不受影響）：第1筆用「SIDE_LINE **中點** ⊥ALLOC」是法定母表「側街臨街線**中點**作垂直線」的用意——鎖死量測起點，避免斜側街時量哪一點都行的模糊。側街 ⊥ 正面時（本案）此值＝`w_1`；斜側街時與 `S·|d_hat·n_alloc|` 會差一點，屆時以「SIDE_LINE 中點 ⊥ALLOC」為準。

## 七、§5a 街廓深度與最小分配面積（沿 ALLOC_LINE 量，KL 澄清新增）

### 法源 + 原則
KL 澄清的最小分配面積定義 + 鐵律 G（一切方向以 ALLOC_LINE 為唯一基準）。深度**必須沿 ALLOC_LINE 方向量**，不可用 MBR（同 W 的修正）。

### 5a-0. 深度權威定義（建築法規版，KL 以「圓弧街廓深度計算範例」鎖定）
> **D_avg = 沿整條 FRONT_LINE 折線、每點對 BASELINE 的垂距，取長度加權積分均值**（線積分均值；KL 的「微積分概念」）。
- FRONT_LINE 是折線（各段 `Ri_Fk`）。對 FRONT 上每一點，作**垂直於 BASELINE 方向**的垂線、量到 BASELINE 的長度＝該點深度（＝該點到 BASELINE 折線的**最短距離**）；整條 FRONT 所有點深度求均值（長段貢獻較多點＝長度加權）＝ `D_avg`。
- **D_min（最小深度）** = 這些垂距的**最小值**（法規：臨路境界線至後側境界線垂直距離之最小值）。供畸零地/最小深度檢查。
- **截角**：角地若需截角，寬度/深度取**截角前**之值。
- **實作**：沿 FRONT 密取樣 → 各點 `P.distance(BASELINE_linestring)`（即 ⊥ BASELINE 最短距）→ 全 FRONT 長度加權平均＝D_avg、取最小＝D_min。需**真實 BASELINE**（4 條、共用者掛兩塊）。

**此定義同時涵蓋直線與弧形**：
- **矩形/近矩形（本案 V6 R1~R6）**：FRONT∥BASELINE、垂距近常數 → 積分均值 ≈ 方法 B（面積÷n_alloc 跨幅），差 <1%（軸向 84.7~87.4°）。**故方法 B 是「矩形捷徑」、本案 R2=43.9 仍成立**。
- **弧形街廓**：垂距沿折線明顯變化 → **方法 B 會錯，必須用本 5a-0 積分均值**（＝「方法 A 的正解」；先前 A 給 500 是合成假 BASELINE 之誤、非垂距概念錯）。

### 5a-1. 街廓深度 `D_block`（本案 V6 用方法 B＝矩形捷徑；弧形改 5a-0 積分均值）

> ⚠️ **W-C 實測修正（取代原「A 主用」決定）**：首版實作把方法 A 設主用，R2 算出 `D_avg=500`（垃圾）。根因＝呼叫端（13678）合成 ±1000m(`_Lb`) BASELINE，R2 那條角度偏 ~90°（幾乎平行深度軸）→ depth(w) 由 0 掃到 1000。連 R3/R4/R6 的 A 也系統性比 B 深 0.5–1m（同一合成線殘差）。方法 A 在乾淨 BASELINE 下本應 = B，故 **A 對「值」零貢獻、只對診斷有用**。R2 left ΣRw=100.02%、W 累積單調 → **alloc_dir（solver 量 W 的軸）已證正確**，500 純污染深度函式、未影響 solver。

**方法 B（面積 / n_alloc 跨幅，深度「值」主用）**——只需街廓多邊形 + alloc_dir、不碰 BASELINE：
```python
ux, uy = f3_cad_alloc_dir[blk]                       # ALLOC_LINE 方向（深度向）
nx, ny = -uy, ux                                     # n_alloc：⊥ ALLOC（臨街向）
proj_n = [p[0]*nx + p[1]*ny for p in 街廓多邊形頂點]
W_block = max(proj_n) - min(proj_n)
D_avg = 街廓面積 / W_block                            # ← 下游 min_alloc_area、① 軟驗證都用這個
```
- 缺 `f3_cad_alloc_dir[blk]` → 警示、不算（不 fallback MBR/area÷front_len）。

**方法 A（剖面差值，降為診斷專用）**——投影 FRONT/BASELINE 取 `depth(w)=|d_base−d_front|`，給 `D_min/D_max` 偵測街廓畫歪：
- **必修**：合成 BASELINE 的 `±1000m`（13678 `_Lb=1000`）→ **裁到街廓 bbox 對角線長**，A 才不會洩 1000。
- A 僅輸出診斷；**值一律取 B**。`|D_avg_A − D_avg_B| > 15%×D_avg_B` → 警示「街廓 X 恐畫歪或 BASELINE 資料異常，已採方法 B」。
- 提示文字修正：把 4937「無 FRONT/BASELINE 折線 → 用方法 B」改為符合實況的「**缺 FRONT/BASELINE 資料**」（其實是資料沒傳進函式、與「折線」無關）。

**取代現行深度計算（§0.5-D3）**：
- 現行 auto = 街廓面積 / 正面長度（除 FRONT_LINE 長，13811/14717）→ 斜街廓偏 → **改方法 B（除 n_alloc 跨幅）**。
- 現行 UI 覆寫（13788 `_alloc_depth_default`）= 全域單一值套全街廓 → **改逐街廓欄位**（每 Ri 一格，留白 = 用算出的 `D_avg_i`），移除「單值套全」。

**資料品質（已由 V6.dxf 實測釐清，見 §0.5-E）**：DXF 的 BASELINE 皆存在且正確（R1、R2/R3 共用、R4、R5/R6 共用 = 4 條），**Claude Code 先前「R1/R5 缺/R2 歪」係誤判**。R2 深度方法 A=500 之真因＝合成 BASELINE 取到共用/錯置線、角度偏 ~90°；改方法 B（不碰 BASELINE）即解。共用 BASELINE 須同時掛給相鄰兩塊（程式修，使用者勿補重疊線）。F 值可當深度旁證（R2 F=44.95 ≈ 真深度 43.9）。

### 5a-4. 交叉驗證：街角範圍 ↔ 最小分配面積（KL 澄清，街角街廓限定）

```

```

> 此式僅街角街廓可用（需街角範圍）；非街角街廓只能走 §5a-1/5a-2（D_avg×面寬）。
> ⚠️ 此式是**交叉驗證**、不是 7-0a 門檻來源；7-0a 門檻一律走 5a-2/5a-3（D_avg×面寬）。現在門檻與此式都用 D_avg，一致。
